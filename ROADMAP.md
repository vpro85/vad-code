# 🗺️ Дорожная карта развития vad-code

## Текущее состояние (v0.4.0)

✅ **Реализовано:**
- Мульти-провайдерная архитектура (OpenAI, Ollama, Anthropic, LM Studio)
- 25+ инструментов для работы с файловой системой
- Git-интеграция (15+ команд)
- Валидация аргументов через Pydantic
- Безопасная работа с ФС (защита от path traversal)
- LRU-кэширование содержимого файлов
- Управление контекстом (обрезка по токенам)
- Retry-логика для LLM-запросов
- Строгая типизация (mypy strict)
- Линтинг (pylint 10/10, flake8)
- Тестовое покрытие (236 тестов)
- Docker-поддержка
- Модульная архитектура инструментов (разделение на специализированные модули)
- Улучшенная обработка ошибок (кастомные исключения, таймауты)
- Новые инструменты: `install_package`, `run_tests`, `format_code`
- Управление проблемными случаями (bad cases)
- Уровни доступа к инструментам (READ/WRITE/DANGEROUS)
- CLI-аргументы для конфигурации
- Сохранение/загрузка истории сессии
- **Система откатов (undo/redo)** — автоматические бэкапы перед изменениями
- **Аудит действий агента** — логирование вызовов инструментов
- **Улучшенная защита `run_command`** — белый/черный список, 20+ опасных паттернов
- **Мониторинг и метрики** — детальная статистика по инструментам, токенам, времени
- **Пресеты конфигураций** — готовые `.env` для популярных моделей
- **Улучшенные сообщения об ошибках** — дружелюбные шаблоны с подсказками

## 🎨 Дизайн-принципы

1. **Безопасность прежде всего** - никаких необратимых действий без подтверждения

4. **Производительность** - минимальное использование токенов
